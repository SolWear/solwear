# Глоссарий

Единый справочник терминов, используемых в документации и коде проекта **Solvare**.

## Термины

**Cold Wallet (холодный кошелёк)**  
Кошелёк, в котором приватный ключ хранится на устройстве, изолированном от постоянного сетевого доступа (здесь — NFC-устройство, например часы); подпись выполняется локально на этом устройстве.

**Hot Wallet (горячий кошелёк / горячий интерфейс)**  
Клиент с доступом к сети (здесь — телефон), который не хранит приватный ключ, но строит транзакции, запрашивает баланс и отправляет данные на подпись холодному устройству.

**NFC (Near Field Communication)**  
Короткодистанционная беспроводная связь между устройствами вблизи друг друга; в Solvare используется для обмена между телефоном и часами.

**NDEF (NFC Data Exchange Format)**  
Стандартный формат записи сообщений NFC; в проекте через NDEF передаются запросы и ответы между приложением на телефоне и прошивкой/приложением на NFC-устройстве.

**Solana**  
Блокчейн-платформа с высокой пропускной способностью; Solvare взаимодействует с ней через RPC для балансов, blockhash и отправки транзакций.

**SOL**  
Нативная монета сети Solana; в MVP приложение работает только с SOL, без SPL-токенов.

**Lamport (лампорт)**  
Наименьшая неделимая единица SOL (как сатоши для биткоина); суммы и комиссии в протоколе Solana часто задаются в лампортах.

**Base58**  
Кодировка для компактного текстового представления бинарных данных (ключи, адреса); в экосистеме Solana адреса обычно отображаются в Base58.

**RPC (Remote Procedure Call)**  
Протокол доступа к узлу сети; приложение вызывает методы Solana JSON-RPC (баланс, recent blockhash, отправка транзакции).

**devnet**  
Тестовая сеть Solana для разработки; в MVP Solvare ориентирован на devnet, токены и транзакции там не имеют реальной стоимости.

**mainnet-beta**  
Основная публичная сеть Solana с реальными активами; в описании MVP явно отделена от devnet как нецелевая среда для первой версии.

**pubkey (публичный ключ)**  
Публичная часть пары ключей ed25519; в Solana служит адресом счёта и передаётся с холодного устройства на телефон для отображения и построения транзакций.

**transaction (транзакция)**  
Подписанный пакет инструкций для сети Solana (например, перевод SOL); телефон собирает сообщение, часы подписывают его приватным ключом.

**signature (подпись)**  
Криптографическая подпись ed25519 над телом транзакции (или соответствующим хэшем), подтверждающая согласие владельца ключа; создаётся на холодном устройстве.

**blockhash**  
Хэш недавнего блока, привязывающий транзакцию ко времени и защищающий от повторного воспроизведения; запрашивается по RPC перед финализацией транзакции.

**SPL token**  
Стандарт токенов Solana (аналог ERC-20); в текущем MVP не поддерживается, в глоссарии упоминается для контекста ограничений.

**CoinGecko**  
Внешний сервис рыночных данных; приложение использует его API для приблизительного отображения стоимости SOL в USD.

**Foreground Dispatch**  
Механизм Android, позволяющий активности перехватывать обнаружение NFC-тега, пока приложение на переднем плане; удобен для сценариев «поднеси часы к телефону» без постоянного системного диалога.

**APDU (Application Protocol Data Unit)**  
Блоки команд и ответов в низкоуровневом протоколе смарт-карт ISO 7816; на практике встречается в документации по NFC/безопасным элементам и отличается от высокоуровневого обмена через NDEF в Solvare.

**ed25519**  
Алгоритм цифровой подписи на эллиптических кривых, используемый в Solana для ключей счетов и подписей транзакций.
